Features

Everything you need,
nothing you don't

One repo. One setup. Target repos stay completely untouched.

🧠

Smart Refinement

The agent reads your codebase, understands the ticket, and decides the optimal approach — pass-through, refine, or decompose.

⚡

Parallel Execution

Complex tickets split into independent subtasks that run simultaneously. Multiple Claude instances create PRs in parallel.

🔒

Guardrails

Built-in validators ensure quality — issues are always created, Jira stays updated, subtask counts are controlled, errors are caught.

🔄

Auto-Sync

The Sync Agent polls every 15 minutes. PR opened → Jira updates. PR merged → ticket is Done. All subtasks done → parent is Done.

🗺️

Multi-Repo Routing

Map Jira components to different GitHub repos with a single JSON config. Add new repos in seconds — no secrets, no workflows.

🏗️

Zero Infrastructure

No servers, no databases, no Docker. Just GitHub Actions and a JSON config. Target repos need nothing installed.

🕸️

Code Knowledge Graph

Onboard a repo and Graphify builds a multi-language knowledge graph — kept fresh on every merge. Agents read it to ground refinement and cut subtasks along real coupling boundaries.

How It Works

From ticket to PR
in minutes, not days

01
📋

Ticket Transitions

Move a Jira ticket to "To Refine". Jira Automation fires a webhook to your DevFlow Kit repo.

02
🔍

Agent Analyzes

The Refinement Agent reads your codebase — CLAUDE.md, README, and an optional Graphify knowledge graph of the repo — and deeply understands the request.

03
🧩

Smart Decision

Simple bug? Straight to implementation. Multi-concern feature? Decomposed into parallel subtasks with non-overlapping file scopes.

04
🚀

Issues Created

GitHub issues are created with @claude tags. Each issue has a full technical spec, scope constraints, and acceptance criteria.

05
🤖

Claude Implements

Claude Code picks up each issue, reads the codebase, implements the changes, and opens a pull request — all fully automated.

06
✅

Jira Auto-Syncs

PRs opened → In Review. PRs merged → Done. All subtasks complete → parent ticket transitions to Done.

Quick Start

Up and running
in 15 minutes

1

Use This Template

Click "Use this template" on GitHub to create your own copy.

git clone https://github.com/your-org/devflow-kit.git
2

Configure Secrets

Add 5 secrets to your repo — Claude token, GitHub PAT, and Jira credentials.

make setup-secrets
3

Map Your Repos

Edit repo-map.json to route Jira projects to GitHub repos.

{
  "routes": [{
    "jira_project": "MYPROJ",
    "component": "backend",
    "github_repo": "your-org/api"
  }]
}
4

Connect Jira

Create one Automation rule in Jira that sends a webhook on ticket transition.

make setup-jira
5

Watch the Magic ✨

Move a Jira ticket to "To Refine" and watch it flow through to a pull request.
